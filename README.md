# Superlight UI

مساحة بسيطة لقراءة محتوى ووردبريس (WordPress) بعيداً عن التعقيد. واجهة أمامية فائقة الخفة تعتمد على البرمجيات الخام (Vanilla JS) لضمان السرعة والتركيز على النص.

## المفهوم الأساسي (The Concept)
الهدف من هذا المشروع هو توفير واجهة مستخدم لا تتجاوز **47 كيلوبايت** عند التحميل الأول. تم تحقيق ذلك من خلال:
- الاستغناء الكامل عن إطارات العمل الثقيلة (No Frameworks).
- استخدام CSS مخصص ومبني بدقة باستخدام Tailwind CLI.
- تضمين الأيقونات (Lucide) برمجياً داخل الكود لتجنب تحميل ملفات خارجية.
- تصفح هادئ، بلا رسوم متحركة، وبلا مشتتات.

## الإمكانيات
- دعم كامل للغة العربية (RTL).
- وضع ليلي (Dark Mode) يحترم اختيار المستخدم.
- وضع "الصور المعطلة" (Minimal Mode) كخيار افتراضي لتوفير البيانات والتركيز على القراءة.
- دعم أنواع المنشورات المخصصة (CPTs).
- نظام ترقيم صفحات (Pagination) يعتمد على التحميل التدريجي (Load More).
- **وضع القارئ (Reader Mode)**: ميزة اختيارية تسمح للمستخدم بتغيير رابط الـ API وإعدادات الموقع يدوياً من داخل الواجهة، مع حفظ الإعدادات في المتصفح.
- **تحميل ملف واحد (Zero Dependencies)**: يتم دمج كافة ملفات CSS و JS داخل ملف HTML واحد عند البناء لتقليل طلبات الشبكة.

## التطوير (Development)
يتطلب المشروع وجود [Bun](https://bun.sh) أو [Node.js] لبناء ملفات التنسيق:

1. قم بتثبيت التبعات (Dependencies):
   ```bash
   bun install
   ```

2. تشغيل الموقع في وضع التطوير (Development):
   ```bash
   bun run dev
   ```
   *يعمل هذا الوضع على ملف `dev-index.html` ويقوم بتحميل `app.js` و `dist.css` كملفات منفصلة لتسهيل التجربة.*

3. بناء النسخة النهائية (Build for Production):
   ```bash
   bun run build
   ```
   *يقوم هذا الأمر بضغط الأكواد ودمجها بالكامل داخل ملف `index.html` واحد.*

## النشر (Deployment)
المشروع مصمم ليعمل كملف واحد (Static File). للنشر، تحتاج فقط لرفع ملف واحد:
- `index.html` (الملف الناتج من عملية البناء `bun run build`)

تأكد من تحديث رابط الـ API في كائن `CONFIG` داخل ملف `app.js`.

## وضع القارئ (Reader Mode)
تم تصميم هذا الوضع لجعل الواجهة عامة وصالحة للاستخدام مع أي موقع ووردبريس:
- **التفعيل**: قم بتغيير قيمة `readerMode` إلى `true` في ملف `app.js`.
- **الاستخدام**: سيظهر زر "إعدادات" (⚙️) في الشريط الجانبي، ومن خلاله يمكن تغيير رابط الـ API، واسم الموقع، وأنواع المحتوى.
- **الحفظ**: يتم تخزين الإعدادات في `localStorage` بالمتصفح، مما يعني أنها ستبقى محفوظة حتى بعد إغلاق الصفحة.

## القيود (Limitations)
- يعتمد بالكامل على JavaScript؛ لذا لن يعمل في حال تعطيل المتصفح للجافاسكريبت.
- جلب المحتوى محدود بـ REST API الخاص بووردبريس (يجب تفعيل الـ CORS في حال كان النطاق مختلفاً).
- لا يدعم حالياً التعليقات بشكل مباشر.
